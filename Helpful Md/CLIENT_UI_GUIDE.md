# Client UI Guide

## Purpose

This guide explains how a client-facing user should move through the calculator without getting overwhelmed by installer terms.

Use `Client Estimate` when the goal is:
- early budget framing
- comparing packages
- understanding what the system will and will not support
- preparing for a clearer installer discussion

The interface is now guided:
- each major section shows a short summary line
- the guide card highlights the recommended next move
- the guide card explains the focused field in plain language while you work
- sections can stay collapsed until they matter

For a plain-language explanation of the result scorecards, use `Helpful Md/OUTCOME_RESULTS_AND_SCORE_GUIDE.md`.

For worked examples across residential, retail/POS, tailoring, bakery, filling station, cold room, workshop, mini-factory, and plant-engineering jobs, use `Helpful Md/INSTALLER_CLIENT_SCENARIO_PLAYBOOK.md`.

The results area is now segmented:
- `System Summary` stays visible first
- `Results Navigator` lets you jump to the next section quickly
- longer sections stay collapsible so the estimate does not feel like one endless report

## Recommended Navigation Order

1. Switch to `Client Estimate`
- This keeps the interface simpler.
- Advanced engineering controls stay hidden.

1.5. Use the `Guided Flow` card as the reading order
- Open the section it recommends next.
- Do not try to fill every section before the load list and project story are honest.
- Collapse completed sections so the active task stays clear.

2. Set the project story first
- `Location Profile`
- `Business Profile`
- `Operating Intent`
- `Continuity Class`
- `Operating Schedule`
- Read the smart conclusion under `System Configuration` after you pick these.

If you are unsure about a term, do not guess aggressively. Use the closest real description of the business.

3. Use a template when the business is familiar
- Good quick starts:
  - retail shop
  - tailoring studio
  - bakery
  - filling station
  - cold room
  - workshop
  - mini-factory

4. Adjust the load list honestly
- Add or remove machines so the draft reflects the real site.
- Use real quantities and realistic usage hours.

5. Read the result in this order
- `Executive Snapshot`
- `Proposal Budget`
- `Commercial Strategy Recommendation`
- `Plant Scope Boundary`
- `Current Working Surface`
- `Supported Load Story`
- `Submission Pack Readiness`
- `Proposal Readiness`

Only after that should you look at deeper technical sections with the installer.

## What The Main Terms Mean

### Business Profile
The type of site being sized.

It helps the calculator choose realistic defaults and warnings.

The interface now adds a smart conclusion if the chosen profile, intent, schedule, or continuity story start to pull away from each other.

### Operating Intent
The broad goal of the system.

Examples:
- backup only
- daytime solar-first
- full off-grid
- hybrid with generator support
- hybrid with grid support

### Continuity Class
How serious an outage is for the business.

Examples:
- `Convenience`: outage is uncomfortable
- `Business Critical`: outage quickly hurts trading
- `Process Critical`: outage disrupts production
- `Product-Loss Critical`: outage risks spoilage or stock loss

### Proposal Readiness
How ready the current draft is to be treated as a real quote.

High readiness does not mean “install tomorrow without survey.” It means the draft is commercially and technically well-controlled for proposal use.

### Submission Pack Readiness
How complete the formal survey, design, approval, and handover pack is for this location and system path.

Clients should read this as:
- how much paperwork and technical closeout still remains
- whether the approval lane is simple or authority-heavy
- whether the installer is still pricing a serious draft or a near-final submission set

### Commercial Strategy Recommendation
The calculator’s honest recommendation about the safest way to describe the job.

Examples:
- battery-dominant off-grid
- solar-first with battery bridge
- hybrid with generator assist
- hybrid with grid support
- essential-load-only backup

This is important because not every business should be sold as a pure battery-only system.

### Plant Scope Boundary
This explains whether the draft is still a normal business-site plant or whether it is becoming a more complex multi-feeder / mini-plant case.

Clients should read this as:
- is this still a captive on-site system?
- is the site getting into a more complex private distribution setup?
- is the job already beyond normal proposal scope and into utility-style approval or plant-study territory?

### Recommended Feeder Schedule
This explains the plant promise in simple operating lanes.

Clients should read it as:

- what is truly on the protected continuity path
- what still depends on generator, grid, timing, or staging
- what is outside the current promise

This is especially useful on larger business-site jobs where one total system size can hide several different operating lanes.

The installer may also use a feeder-brief export from this same result when the job needs feeder-level follow-through behind the proposal.

### Current Working Surface
This tells the client whether the draft still reads like a normal site proposal or whether it has crossed into a heavier plant-engineering conversation.

Clients should read it as:

- is this still a normal commercial proposal surface?
- is the installer now treating it like a plant handoff?
- does the job need a separate utility / mini-grid engineering path before final commitment?

### Supported Load Story
This explains what the proposal is really promising.

Read it this way:

- `Protected Path`: the loads the quote is treating as covered continuity
- `Assisted Path`: loads that still need generator help, grid help, daylight timing, or staging
- `Outside The Promised Path`: loads that are not included in the continuity promise unless the system scope changes

This section matters because a good quote is not only about total size. It is about being clear on what the system will reliably carry.

### Commercial Value Outlook
This now shows two layers:

- the short sales view: annual value and simple payback
- the medium-term caution view: annual maintenance plus planned inverter and battery refresh allowances
- the optional finance-sensitivity view: debt service, tax benefit, and residual value when the installer opens the advanced-finance section
- the optional scenario view: cash purchase versus financed purchase when the installer models financing

Read both together. A good simple payback headline can still need a more careful lifecycle explanation on battery-heavy projects.

If the installer shows a financed version of the project, treat that as a sensitivity view, not a bank-approved offer.

### Commercial Preset
This is the installer's reusable quote posture.

It can switch the proposal toward:
- value-led
- balanced standard
- continuity-heavy
- premium
- financed comparison

Clients should read it as a starting commercial posture, not as proof that the site survey, supplier quote, or financing has already been finalized.

### Supplier Quote Freshness
This tells you how current the supplier-backed pricing story really is.

Read it this way:

- `Benchmark-only` means the quote is still an informed estimate
- `Fresh live quote` means the installer has dated supplier pricing inside the current refresh window
- `Aging` or `Stale` means the price story should be refreshed before being treated as final

If the quote looks strong technically but the supplier quote freshness is weak, treat the budget as a scoping figure rather than a locked commercial commitment.

The installer workflow now also includes a supplier refresh brief, so aging or stale quotes can be re-asked against the same design basis instead of being rebuilt loosely by hand.

### Supplier Quote Import
Clients do not need to operate this directly, but it matters because it improves commercial traceability.

It helps the installer turn pasted or file-based supplier quote data into:

- dated quote metadata
- clearer supplier references
- more consistent component-rate overrides

That makes the proposal easier to audit later.

## How To Use The Result Properly

### Read the headline first
The `Executive Snapshot` tells you:
- what kind of system story is being recommended
- how ready the proposal is
- what assumptions still need confirmation

### Read the budget second
The `Proposal Budget` helps you understand:
- approximate pricing posture
- package comparison
- scope boundaries

Use `Results Navigator` if the page feels long. It is there to reduce scrolling, not to hide information.

### Read the strategy third
If the system recommends `hybrid with generator assist` or `essential-load-only backup`, that is not a downgrade. It is usually a more honest business answer.

### Use the outcome guide when percentages look strong
If `Confidence`, `Proposal Readiness`, `Compliance`, or `Submission Pack` look high, that still does not mean the job skips survey, approvals, or installer review. Use `Helpful Md/OUTCOME_RESULTS_AND_SCORE_GUIDE.md` when you need the plain-language meaning of each score.

## Common Misreadings To Avoid

- Do not assume bigger battery always means a better project.
- Do not assume a filling station, bakery, or workshop should automatically be full off-grid.
- Do not assume every business needs three-phase.
- Do not assume every “mini solar plant” request is the same as a utility mini-grid.
- Do not assume every template already matches your exact machine list.
- Do not treat a preliminary estimate as a final surveyed design.

## Practical Examples

### Tailoring studio
Often starts as single-phase.

The real questions are:
- servo or clutch machines
- ironing load
- number of machines running together

### Bakery
The first real question is oven type:
- gas-assisted or diesel-assisted may stay lighter
- fully electric oven can push the project into a harder topology

### Filling station
This is often about selective continuity and assisted support, not long deep battery autonomy for the whole site.

### Cold room
The overnight preservation burden is usually more important than daytime convenience loads.

## Best Client Workflow

Use client mode to answer:
- what am I really trying to protect?
- what system posture is realistic?
- what package level matches the budget?
- what survey items still affect the final quote?

Then hand the job to installer mode for:
- phase review
- board strategy
- inverter cluster checks
- battery throughput checks
- generator path checks
- submission-pack closeout
- final technical sign-off

## Simple Navigation Rule

Keep this order in mind:
- `Site Setup`
- `Project Draft`
- `Survey & Quote`
- `Loads`
- `Proposal Finish`

That sequence is more reliable than jumping straight to price or battery size.
