# Authority Submission Packs Guide

## Purpose

This guide explains the staged `Submission Pack Readiness` layer in the calculator.

It does not claim to replace stamped engineering, jurisdictional permit advice, or OEM approval rules.
It exists to make the transition from sizing/proposal work to real closeout work more structured and less error-prone.

## What The Feature Does

For the selected:

- location profile
- system type
- survey state
- proposal identity state
- equipment / design state

the app now derives a structured submission pack with four stages:

1. `Survey and intake pack`
2. `Technical design dossier`
3. `Approval lane`
4. `Commissioning and handover pack`

Each stage is shown with:

- `Closed`
- `Pending`
- `Open`

and the panel also tracks:

- governing approval lane
- focused deliverables
- open submission actions
- pack notes

## What The Feature Does Not Do

It does not:

- generate stamped authority forms
- guarantee permit acceptance
- replace local engineer / electrician responsibility
- replace final protection, interconnection, or OEM approval review
- certify that a site is legally install-ready

## Region Mapping

The calculator now maps each region to a structured submission profile instead of using only a generic compliance note.

Examples:

- `Lagos / Nairobi / Accra / Brazil`
  - licensed installer closeout pack
- `US South / US North`
  - AHJ permit and interconnection pack
- `EU Central / EU South`
  - DNO / grid-operator pack
- `India`
  - DISCOM and installer submission pack
- `UAE`
  - approved-contractor utility pack
- `Australia`
  - DNSP / retailer submission pack

## How To Read The Four Stages

### 1. Survey and Intake Pack

This stage checks whether the job has enough commercial identity and survey kickoff structure to support a formal cover pack.

Typical items:

- installer company
- lead contact
- client / site identity
- quote reference and issue date
- survey kickoff evidence

### 2. Technical Design Dossier

This stage checks whether the app has enough design substance to support a real technical file.

Typical items:

- single-line diagram
- load schedule / design basis
- equipment datasheets or cut sheets
- array layout
- protection summary

### 3. Approval Lane

This stage changes by system path.

For `Off-Grid` jobs it usually behaves like installer electrical sign-off and DB isolation closeout.

For `Hybrid` and `Grid-Tie` jobs it behaves like a true authority / utility / interconnection lane.

Typical items:

- installer electrical sign-off
- utility or authority application trail
- anti-backfeed / export / operating-mode declaration
- approved equipment declaration where applicable

### 4. Commissioning and Handover Pack

This stage checks whether the promised scope, compliance posture, and closeout material are aligned enough for handover.

Typical items:

- commissioning checklist
- settings record
- closeout record
- handover note

## Practical Interpretation

### When the pack is `Submission-ready`

It means:

- the structured paperwork story is largely aligned
- the job is closer to a real submission or handover package
- installer closeout discipline is in place

It does **not** mean:

- the authority has approved anything
- the design is stamped
- no more engineering judgment is needed

### When the pack is `In progress`

It means the project is serious, but there are still material gaps in survey, approval lane, or handover controls.

This is a normal state for many live quotes.

### When the pack is `Open`

It means the project should still be treated as a scoping or internal-review package, not a formal submission or closeout set.

## Best Use By Audience

### Installer

Use it to avoid these mistakes:

- sending quotes with weak survey control
- promising authority-ready work without the right approval lane
- forgetting handover / commissioning closeout
- treating generic equipment placeholders like a finalized dossier

### Client

Use it to understand:

- whether the job is still an estimate or close to formal closeout
- whether the project needs utility or authority interaction
- why some jobs need more paperwork before installation

## Honest Boundary

This feature materially improves commercial discipline and proposal honesty.

It is still a preparation and readiness layer, not a substitute for:

- local code interpretation
- stamped engineering
- authority submission ownership
- OEM parallel-rule validation
- final field commissioning sign-off
